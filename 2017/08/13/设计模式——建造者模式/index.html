创建型模式

  2. 工厂模式

建造者模式正是其中一种

建造者模式

简介/动机

动机

将产品内部表象（细节）与怎么构建的具体过程分离，从而使同样的构建过程生产具有不同表象的产品。

产品的内部表象： 产品通常由不同零件组成。这种零件可以是对象，或者成员变量，通常被叫做产品的内部表象。不同的产品有不同的表象（零件），建造者模式让客户端不需要知道产品由哪些零件组成，怎么建造，怎么组成。

解决

主要解决在开发过程中，可能遇到一个复杂产品的创建。该产品的构建需要一系列的内部表象（零件或对象）通过固定的算法构成，因为需求盖面，内部表象既对象存在着大量变化，但是将它们组合在一起的算法确实相对固定的。

角色

  1. 构建者接口（Builder）: 独立于业务层外，规范对产品内部表象（对象）建造的方法表示。直接建造产品具体内部表象（对象）是具体构建者。具体构建者需要实现该接口的所有方法。一般来说，方法的数量和内部表象（对象或零件）的数量一一对应，也就是有多少零件，就有多少建造方法。除了构造零件的方法外，还有一种返回构建结果的抽象方法。
  2. 具体构建者（ConcretBuilder)：该角色的任务有两种。1、实现构建者接口的方法，描述零件的建造步骤。2、构建完成后，向客户端提供构产品实例。
  3. 导演者（Director)：该角色调用具体构建者创建产品对象。需要知道的是，导演者角色扮演一种指挥的作用，它并没有产品的任何知识，这些知识包含在具体构建者角色中。
  4. 产品（Product）： 需要生产复杂产品对象。

需要注意的是，客户端主要与导演者角色打交道。首先，客户端将创建产品对象的请求交与导演者。导演者将请求划分为不同的构建步骤交与具体构建者。具体构建者对于客户端来说，是隐藏的。
一般而言，一个产品对应一个构建者角色，产品的每一个零件对应于构建者角色中的每一个构建方法。
